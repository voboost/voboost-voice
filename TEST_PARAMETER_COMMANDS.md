## 📋 КОМАНДЫ С ПАРАМЕТРАМИ

### 1️⃣ Звонок контакту

**Шаблон:**
```json
{
  "id": "phone_call_contact",
  "patterns": [
    "позвони {contact}",
    "вызови {contact}",
    "набери {contact}",
    "звонок {contact}"
  ]
}
```

**Примеры:**
```
"позвони жене"        → contact = "жене"
"вызови маму"         → contact = "маму"
"набери иван"         → contact = "иван"
"звонок работа"       → contact = "работа"
```

**NLUEngine извлечёт:**
```kotlin
"позвони жене" → { "contact": "жене" }
"вызови маму"  → { "contact": "маму" }
```

**Intent будет отправлен:**
```kotlin
intent.putExtra("voice.param.contact", "жене")
intent.putExtra("voice.param.call_type", "contact")
```

## ⚠️ ВОЗМОЖНЫЕ ПРОБЛЕМЫ

### 1. Распознавание чисел

**Проблема:**
```
"22" может быть распознано как:
- "двадцать два"
- "два два"
- "двадцать"
```

**Решение:**
Добавить нормализацию чисел в NLUEngine:

```kotlin
private fun normalizeNumber(text: String): String {
    return text
        .replace("двадцать", "20")
        .replace("тридцать", "30")
        .replace("сорок", "40")
        .replace("пятьдесят", "50")
        // и т.д.
}
```

### 2. Падежи имён

**Проблема:**
```
"жена" → "жене" (дательный падеж)
"мама" → "маме" (дательный падеж)
```

**Решение:**
Использовать поиск в телефонной книге по нормализованному имени.

### 3. Длинные команды

**Проблема:**
```
"позвони жене по мобильному" → contact = "жене по мобильному"
```

**Решение:**
Уточнить шаблоны:
```json
"patterns": [
    "позвони {contact}",
    "позвони {contact} по телефону",
    "вызови {contact}"
]
```
