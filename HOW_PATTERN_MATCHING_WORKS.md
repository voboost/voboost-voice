## ❓ ВОПРОС

> "В конфиге нет команды 'позвони жене'. Как мы понимаем что эта фраза относится к шаблону `позвони {contact}`?"

---

## ✅ ОТВЕТ

Мы **НЕ ищем фразу в конфиге**! Вместо этого мы:

1. **Берём шаблон из конфига** → `"позвони {contact}"`
2. **Преобразуем в regex** → `"^позвони\s+(.+)$"`
3. **Проверяем фразу на соответствие regex** → `"позвони жене"` ✓
4. **Извлекаем параметр из группы** → `"жене"`

---

## 📊 ПОШАГОВЫЙ АЛГОРИТМ

### Шаг 1: Загрузка конфига

```json
{
  "id": "phone_call_contact",
  "patterns": [
    "позвони {contact}",
    "вызови {contact}",
    "набери {contact}",
    "звонок {contact}"
  ]
}
```

**Что в конфиге:**
- ✅ Шаблоны с параметрами `{contact}`
- ❌ Конкретных имён ("жене", "мама", "иван") НЕТ

---

### Шаг 2: Получение распознанной фразы

```kotlin
val recognizedText = "позвони жене"
```

**Что распознал Sherpa/Vosk:**
- ✅ Фраза пользователя: "позвони жене"
- ❌ Такого шаблона в конфиге НЕТ (есть только "позвони {contact}")

---

### Шаг 3: Перебор всех шаблонов

```kotlin
for (commandConfig in commands) {
    for (pattern in commandConfig.patterns) {
        val matchResult = matchPattern(recognizedText, pattern)
        
        if (matchResult != null) {
            // Команда найдена!
            return RecognizedCommand(...)
        }
    }
}
```

**Процесс:**
```
Итерация 1: pattern = "открой лючок зарядки"
  matchPattern("позвони жене", "открой лючок зарядки")
  → false (не совпадает)

Итерация 2: pattern = "позвони {contact}"
  matchPattern("позвони жене", "позвони {contact}")
  → true (совпадает!) ← НАШЛИ!
```

---

### Шаг 4: Преобразование шаблона в regex

**Функция `buildRegex()`:**

```kotlin
private fun buildRegex(pattern: String): String {
    val escapedPattern = pattern
        .lowercase()
        .trim()
        .replace("{", "(.+)")  // Заменяем {contact} на (.+)
        .replace("}", "")
        .replace(" ", "\\s+")  // Пробелы -> \s+
    
    return "^$escapedPattern$"
}
```

**Пример:**
```
Вход:  "позвони {contact}"
Шаг 1: "позвони {contact}" → lowercase()
Шаг 2: "позвони (.+)" → replace("{", "(.+)")
Шаг 3: "позвони (.+)" → replace("}", "")
Шаг 4: "позвони\s+(.+)" → replace(" ", "\\s+")
Шаг 5: "^позвони\s+(.+)$" → добавить ^ и $

Результат: "^позвони\s+(.+)$"
```

---

### Шаг 5: Сопоставление фразы с regex

**Код:**
```kotlin
val regexPattern = "^позвони\s+(.+)$"
val regex = Pattern.compile(regexPattern, Pattern.CASE_INSENSITIVE)
val matcher = regex.matcher("позвони жене")

if (matcher.matches()) {
    // Фраза соответствует шаблону!
    val contact = matcher.group(1)  // "жене"
    return mapOf("contact" to contact)
}
```

**Визуализация:**
```
Фраза:   п  о  з  в  о  н  и     ж  е  н  е
         │                    │  │  │  │  │
Regex:   ^  п  о  з  в  о  н  и \s+ (.+)  $
         │                    │     │
         └────Совпадает────────┘     └─Группа 1: "жене"
```

**Результат:**
```
matcher.matches() → true
matcher.group(1) → "жене"
```

---

### Шаг 6: Извлечение имени параметра

**Функция `extractParamNames()`:**

```kotlin
private fun extractParamNames(pattern: String): List<String> {
    return "\\{([^}]+)\\}".toRegex()
        .findAll(pattern)
        .map { it.groupValues[1] }
        .toList()
}
```

**Пример:**
```
Вход: "позвони {contact}"
Находки: ["contact"]
Результат: ["contact"]
```

---

### Шаг 7: Сопоставление группы с именем

**Код:**
```kotlin
val paramNames = ["contact"]  // Из extractParamNames()
val matcher = regex.matcher("позвони жене")

for ((index, paramName) in paramNames.withIndex()) {
    val value = matcher.group(index + 1)  // Группа 1
    params[paramName] = value  // params["contact"] = "жене"
}
```

**Результат:**
```kotlin
params = mapOf("contact" to "жене")
```

return RecognizedCommand(

    config = commandConfig,

## 🔑 КЛЮЧЕВЫЕ МОМЕНТЫ

| Момент | Объяснение |
|--------|------------|
| **Шаблоны в конфиге** | Содержат `{param}`, а не конкретные значения |
| **Regex** | `{param}` → `(.+)` (любой текст) |
| **Сопоставление** | Проверяем всю фразу на соответствие regex |
| **Параметры** | Извлекаем из захватывающих групп `(.+)` |
| **Гибкость** | Один шаблон работает для любых значений параметра |

---

## ✅ ВЫВОД

**Мы НЕ ищем фразу в конфиге!**

**Мы преобразуем шаблон из конфига в regex и проверяем фразу на соответствие этому regex.**

Это позволяет одному шаблону `"позвони {contact}"` работать для ЛЮБОГО контакта:
- "позвони жене" ✓
- "позвони маме" ✓
- "позвони ивану" ✓
- "позвони на работу" ✓

**Регулярные выражения + захватывающие группы = магия! 🎯**
