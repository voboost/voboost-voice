# Анализ архитектуры TTS-сервиса в `com/qinggan/ttsservice`

## Обзор

Система поддерживает **множество TTS-движков одновременно**, с возможностью переключения в рантайме.  
Используется шаблон **Strategy + Factory**, где:
- Все движки реализуют общий интерфейс через `AbstractTtsImpl`
- `ImplFactory` выбирает активный движок на основе настроек
- `TtsService` — Android-сервис, управляющий жизненным циклом синтеза

---

## Ключевые компоненты

### 1. `TtsService.java`
- **Тип**: `android.app.Service`
- **Назначение**: Центральный TTS-сервис, обрабатывающий запросы
- **Компоненты**:
  - `RequestQueue` — очередь запросов синтеза
  - `TtsImplPool` — пул реализаций движков ( Singleton / pool )
  - `media player manager` — управляет воспроизведением

### 2. `ImplFactory.java`
- **Тип**: фабрика движков
- **Роль**:
  - Выбирает TTS-движок по текущим настройкам (хранятся в SQLite)
  - Поддерживает **гибридные реализации**:
    - `NuanceXiaoIceImpl`
    - `AISpeechXiaoIceImpl`
    - `ArkLitePateoTtsImpl`
    - и др.

### 3. Реализации движков (`impl/`)
| Папка | Движок | Особенности |
|-------|--------|-------------|
| `impl/baidu/` | Baidu TTS | `BaiduTtsImpl.java`, `BaiduUtil.java` |
| `impl/iflytek/` | iFLYTEK (科大讯飞) | Native-биндинг (`IflytekNativeImp.java`), `ITTSPlayListener` |
| `impl/microsoft/` | MS Speech SDK | Полная поддержка SDK v1.x, обработчики ошибок |
| `impl/aispeech/` | AiSpeech | `AISpeechTtsImpl`, аутентификация (`AISpeechAuth`) |
| `impl/arklite/` | ArkLite | Собственный движок (Pateo), PCM-настройки, плейер |
| `impl/xiaoice/` | XiaoIce | Microsoft-совместимый, но кастомная реализация |
| `impl/pateo/` | Pateo | Родной движок для Geely/Pateo |
| `impl/localaudio/` | Локальные файлы | Не требует интернета, `AudioTrack` / `MediaPlayer` |
| `impl/combine/` | Гибриды | Например, Nuance → распознавание, XiaoIce → синтез |

### 4. Кэширование
- `impl/cache/CacheTtsImpl.java` — кэширует результаты синтеза
- Метаданные в `sql/TtsImplSetting.java`

### 5. Отладка и дампы
- `sde/TtsSdeMgr.java`, `sde/TtsDebugReceiver.java`
- `util/QGTtsPcmDump.java` — запись PCM-данных в `/sdcard/` (включается/выключается)

### 6. Настройки и UI
- `sql/TtsSettingsHelper.java`, `TtsContentProvider.java`, `TtsObserverSettingsHelper.java`
- `ui/EngineSettings.java` — экран переключения движка (возможно, в настройках приложения)

### 7. Вспомогательные утилиты
- `util/`:
  - `Config.java` — глобальные настройки
  - `DeviceIdUtils.java` — ID устройства
  - `HttpDownloader.java` — загрузка ресурсов (например, голосов)
  - `TtsFixHelper.java` — хаки для багов на определённых устройствах
- `platform/PlatformAdaptMgr.java` — адаптация под платформу ( Automotive / Android )

---

## Потенциальные «точки входа» для модификации

| Файл | Возможность |
|------|------------|
| `ImplFactory.java` | Изменить логику выбора движка (по умолчанию — `XiaoIce`, `Pateo`, `Baidu` и т.д.) |
| `TtsService.java` | Кастомизация обработки запросов, логгирование, добавление webhook'ов |
| `CacheTtsImpl.java` | Отключить/перенастроить кэш (например, временно — для теста) |
| `QGTtsPcmDump.java` | Включить дамп для отладки артефактов/шумов |
| `util/Config.java` | Включить режим дебага (`LOGD`, `DEBUG_PCM`) |

## Структура данных

### `TtsSpeechBean.java`
- DTO-класс: текст, язык, голос, скорость, высота, ID запроса и т.д.

### База SQLite
- `sql/TtsImplSetting.java` — таблица текущих настроек движка
- `sql/SettingsSqlHelper.java` — CREATE TABLE/UPDATE

## Поддерживаемые функции
- ✅ Multi-engine selection
- ✅ кэширование и офлайн-режим (`localaudio`)
- ✅ гибридные движки (например, Nuance CRP + XiaoIce TTS)
- ✅ отладочная запись PCM
- ✅ сетевые зависимости (авторизация, краудсорсинг голосов)
- ✅ кастомный `AudioTrack`-плейер (низколатентный)

## Заключение

Это **Enterprise-grade TTS-система** с высокой степенью кастомизации и отказоустойчивостью.  
Используется в автомобильных системах (Pateo/Geely), что объясняет:
- Наличие локального движка (`localaudio`)
- Адаптацию под Automotive (`PlatformAdaptMgr`)
- Поддержку offline и fallback (`DummyTtsImpl`)
- Детальную отладку (`QGTtsPcmDump`, `TtsFixHelper`)
